Внимание

Правила валидации

Добавляйте атрибут data-validator="rule1,rule2:param" к полям ввода.

Правило Описание
Основные
required Обязательное поле (не пустое).
required_with:foo Обязательно, если поле foo заполнено.
required_without:foo Обязательно, если поле foo пустое.
Сравнение и логика
eq:val / ne:val Равно / Не равно значению.
gt:N / lt:N Больше / Меньше числа N (или длины строки).
gte:N / lte:N Больше-равно / Меньше-равно.
oneof:a,b,c Значение должно быть одним из списка.
eqfield:name Значение равно значению поля name (например, повтор пароля).
nefield:name Значение не равно полю name.
gtfield:name Значение больше, чем в поле name.
Числа и длина
numeric Только цифры (строка).
number Любое валидное число (включая float и отрицательные).
len:N Длина строки (или массива) ровно N.
min:N / max:N Мин/Макс значение числа или длина строки.
min_alpha:N Минимум N букв (A-Z).
min_lower:N Минимум N строчных букв.
min_upper:N Минимум N заглавных букв.
min_digit:N Минимум N цифр.
min_symbol:N Минимум N спецсимволов (!@#$...).
Строки и формат
alpha Только латинские буквы.
alphanum Буквы и цифры.
alphaunicode Буквы (включая кириллицу и другие алфавиты).
lowercase / uppercase Только строчные / только заглавные.
contains:str Должно содержать подстроку str.
startswith:str Начинается с str.
email Валидный Email адрес.
json Валидная JSON строка.
base64 / base64url Строка в формате Base64.
hexcolor Цвет HEX (напр. #f00 или #ff0000).
Контакты и данные
phone Телефон (мягкая проверка, допускает скобки, дефисы).
e164 Строгий международный телефон (напр. +79990000000).
credit_card Номер банковской карты (алгоритм Луна).
bic Банковский SWIFT код.
uuid / uuid4 UUID идентификатор.
btc_addr / eth_addr Адрес кошелька Bitcoin / Ethereum.
Сеть и интернет
url / http_url URL адрес (http_url требует протокол http/s).
ip / ipv4 / ipv6 IP адрес.
cidr CIDR подсеть (напр. 192.168.1.0/24).
mac MAC адрес.
hostname Имя хоста (домен).
Дата и файлы
datetime Валидная дата (JS Date.parse).
datetime:layout Дата в формате Go Layout.
Пример: datetime:2006-01-02 (ГГГГ-ММ-ДД).
timezone Валидная временная зона (IANA).
ext:jpg;png Расширение файла (для input type=file).
image Файл является изображением.

Кастомные валидаторы

Вы можете расширять функционал библиотеки, добавляя собственные функции проверки для конкретного экземпляра формы. Правила добавляются напрямую в объект validator.rules.

JavaScript
const kris = new KrisForm(document.getElementById('form-custom'));

// 1. Добавляем правило 'magic_code'
kris.validator.rules.magic_code = function(value, param, el) {
    // value - текущее значение поля
    // param - параметр из HTML (в примере: "777")
    // el - DOM элемент поля (удобно для проверки атрибутов)
    
    return value === param;
};

// 2. Добавляем текст ошибки (опционально)
// Если не добавить, будет показано стандартное сообщение
window.KrisFormTranslateMessages.magic_code = "Неверный код. Ожидается: %s";

Сигнатура функции

(value, param, element) => boolean
  • value: Значение поля (всегда строка, если это не чекбокс/файл).
  • param: Строка параметра, переданная после двоеточия (rule:param). Если параметра нет — null.
  • element: Ссылка на DOM-элемент input, select или textarea.
  • return: Верните true, если валидация прошла успешно. Верните false для ошибки.

JSON Зависимости

Структура объекта для управления логикой формы.

Свойство Описание
source name поля, за которым следим.
condition Условие JS (строка). Например: value === 'admin'.
target Список полей (через запятую), на которые влияем.
action Действие при выполнении условия (True).
inverse_action Действие при невыполнении (False).
data-url URL для загрузки данных. Поддерживает плейсхолдер {value}.
message Текст сообщения (можно HTML) рядом с целью.
confirm Текст для модального окна подтверждения.

Список действий (Actions)

show / hide enable / disable required / optional check / uncheck clear set_value:X add_class:cls

Синтаксис выражений и математика

Используется в поле condition (для логики) и в действии set_computed_value (для вычислений).

  • value — значение поля source.
  • fields.OtherName — значение другого поля формы.
  • source.checked — (для чекбоксов) true/false.
  • Логика: ===, !==, >, <, &&, ||.
  • Математика: +, -, *, /, max(...), min(...).

1. Сложная логика

Пример: Выбор "Компании" показывает поле ИНН и делает его обязательным.

HTML Разметка
<form id="form-complex">
  <select name="type" class="form-select">
    <option value="person">Физическое лицо</option>
    <option value="company">Компания</option>
  </select>

  <!-- Контейнер ИНН (скрыт по умолчанию) -->
  <div id="cont_tax" class="d-none mt-3">
    <label>ИНН</label>
    <input name="tax" class="form-control" data-validator="numeric,len:10">
    <div class="invalid-feedback"></div>
    <!-- Место для сообщения зависимости -->
    <div class="field-status-message small text-primary"></div>
  </div>
</form>
JavaScript
const rules = [
  {
    "source": "type", "condition": "value === 'company'",
    "action": "show", "inverse_action": "hide", "target": "tax"
  },
  {
    "source": "type", "condition": "value === 'company'",
    "action": "required", "inverse_action": "optional", "target": "tax",
    "message": "Для юр. лиц ИНН обязателен"
  }
];
new KrisForm(document.getElementById('form-complex'), { dependencies: rules });

Библиотека поддерживает подтверждение действий. Вы можете использовать Bootstrap или свою верстку (Vanilla).

Bootstrap 5
Ванильный JS
HTML (В конце body)
<!-- Для Bootstrap: Стандартная разметка -->
<div class="modal fade" id="modal-bs" tabindex="-1">
  <div class="modal-dialog">
    <div class="modal-content">
      <div class="modal-body">
        <p class="confirm-message"></p>
      </div>
      <div class="modal-footer">
        <button class="btn btn-secondary btn-cancel">Отмена</button>
        <button class="btn btn-primary btn-confirm">ОК</button>
      </div>

<!-- Для Vanilla: Кастомная верстка -->
<div id="modal-vanilla" style="display:none">
  <div class="my-custom-dialog">
    <p class="confirm-message"></p>
    <button class="btn-cancel">Нет</button>
    <button class="btn-confirm">Да</button>

// Конфигурация для Bootstrap формы

  modalId: 'modal-bs', // ID модального окна
  dependencies: [{
    source: "bs_sw", condition: "checked === true", 
    action: "enable", inverse_action: "disable", target: "bs_data",
    confirm: "Включить режим администратора?" // Текст подтверждения
  }]

    source: "v_sw", condition: "checked === true",

  }]

3. Локализация (Перевод ошибок)

Вы можете глобально переопределить тексты ошибок через объект window.KrisFormTranslateMessages. Нажмите на кнопки ниже и попробуйте ввести некорректные значения, чтобы увидеть разницу.

HTML
<form id="form-i18n">
  <label>Возраст</label>
  <input type="number" name="age" data-validator="required,min:18">
</form>
JavaScript
// Глобальная настройка словаря
window.KrisFormTranslateMessages = {
  required: "Это поле обязательно",
  min: "Минимальное значение: %s"
};

// Динамическая смена языка (пример)
function setLanguage(lang) {
  if (lang === 'en') {
    window.KrisFormTranslateMessages = { required: "Required", min: "Min: %s" };
  } else {
    window.KrisFormTranslateMessages = { required: "Обязательно", min: "Мин: %s" };
  }
}

new KrisForm(document.getElementById('form-i18n'));

4. Вычисляемые значения

Используйте действие set_computed_value для выполнения арифметических операций на основе значений других полей. Поддерживаются операторы + - * / и функции min/max.

Автоматически увеличивается, если требования превышают это значение.
JavaScript
// Логика: Если (Цифры + Символы) > Длина, обновить Длину.
const autoFixAction = "set_computed_value:max(fields.cfg_length, fields.cfg_digits + fields.cfg_symbols)";

new KrisForm(document.getElementById('form-computed'), {
    dependencies: [
        // Следим за полями цифр и символов
        { source: "cfg_length", condition: "true", action: autoFixAction, target: "cfg_length" },
        { source: "cfg_digits", condition: "true", action: autoFixAction, target: "cfg_length" },
        { source: "cfg_symbols", condition: "true", action: autoFixAction, target: "cfg_length" }
    ]
});

5. Динамическая загрузка (AJAX)

KrisForm может автоматически загружать данные из API и заполнять поле (Select/Input) при выполнении условий.
Используйте свойство data-url. Плейсхолдер {value} заменяется значением поля-источника.

Источник: jsonplaceholder.typicode.com
JavaScript
new KrisForm(document.getElementById('form-data'), {
    // 1. Базовый URL
    endpoint: "https://jsonplaceholder.typicode.com/",
    
    dependencies: [
        {
            source: "user_id",
            // 2. Условие: Пользователь выбран
            condition: "value !== ''",
            
            // 3. Действие: Включить поле и загрузить данные
            action: "enable",
            inverse_action: "disable",
            target: "albums",
            
            // 4. URL данных (относительно endpoint). {value} заменяется на ID юзера.
            // Итог: https://jsonplaceholder.typicode.com/albums?userId=1
            "data-url": "albums?userId={value}" 
        }
    ]
});
Формат ответа Бэкенда

Библиотека автоматически разбирает JSON в зависимости от типа целевого поля:

  • Для <select>: Ожидается массив объектов.
    • Значение (Value): Берется из поля value или id.
    • Текст (Label): Берется из поля label, name, title или value.
    Пример: [{"id": 1, "name": "Опция А"}, {"value": "b", "label": "Опция Б"}]
  • Для <input>: Ожидается одиночное значение (Строка/Число) или объект с полем value.
Восстановление сохраненных значений

При динамической загрузке через AJAX поле изначально пустое. Чтобы автоматически выбрать сохраненное значение после загрузки данных, добавьте атрибут data-value в HTML: 

<!-- Библиотека выберет 'msk', как только данные загрузятся -->
<select name="city" data-value="msk"></select>
