Getting Started

A pure JavaScript library for form management. Validation and targeted actions on form elements.

<script src="forms.min.js"></script>
<script>
    // Initialization on form element
    new KrisForm(document.getElementById('my-form'));
</script>

Configuration

const options = {
    updateDelay: 10, // Validation debounce (ms)
    modalId: 'dependencyConfirmModal', // Default modal ID
    
    // System message texts
    i18n: {
        defaultError: "Validation failed",
        confirmTitle: "Confirmation",
        confirm: "Yes",
        cancel: "Cancel"
    },
    
    dependencies: [] // Logic array
};

Validation Rules

Add the data-validator="rule1,rule2:param" attribute to your input fields.

Rule Description
Basic
required Field must not be empty.
required_with:foo Required if field foo is present.
required_without:foo Required if field foo is empty.
Logic & Comparison
eq:val / ne:val Equal / Not Equal to value.
gt:N / lt:N Greater / Less than N (number) or length (string).
gte:N / lte:N Greater/Equal / Less/Equal.
oneof:a,b,c Value must be one of the listed.
eqfield:name Equal to value of field name (e.g. Confirm Password).
nefield:name Not equal to field name.
gtfield:name Value greater than field name.
Numeric & Length
numeric Digits only (string).
number Any valid number (integer/float/negative).
len:N String length (or array size) exactly N.
min:N / max:N Min/Max value (number) or length (string).
min_alpha:N Min N alphabetic chars.
min_upper:N Min N uppercase chars.
min_digit:N Min N digits.
min_symbol:N Min N special symbols.
String & Format
alpha Alphabetic characters only.
alphanum Alphanumeric characters.
alphaunicode Letters (including Unicode/Cyrillic).
lowercase / uppercase Lowercase only / Uppercase only.
contains:str Must contain substring str.
startswith:str Must start with str.
email Valid Email address.
json Valid JSON string.
base64 / base64url Base64 encoded string.
hexcolor HEX Color (e.g. #f00 or #ff0000).
Contact & Identity
phone Phone number (loose check, allows dashes/spaces).
e164 Strict international phone (e.g. +19990000000).
credit_card Credit Card number (Luhn algorithm).
bic SWIFT/BIC code.
uuid / uuid4 UUID identifier.
btc_addr / eth_addr Bitcoin / Ethereum wallet address.
Network
url / http_url URL address (http_url requires http/s protocol).
ip / ipv4 / ipv6 IP Address.
cidr CIDR Subnet (e.g. 192.168.1.0/24).
mac MAC Address.
hostname Hostname (domain).
Date & File
datetime Valid date (JS Date.parse).
datetime:layout Date in Go Layout.
Ex: datetime:2006-01-02 (YYYY-MM-DD).
timezone Valid IANA Timezone.
ext:jpg;png File extension (for input type=file).
image File is an image.

Custom Validators

You can extend the library by adding custom check functions to a specific form instance. Rules are added directly to the validator.rules object.

JavaScript
const kris = new KrisForm(document.getElementById('form-custom'));

// 1. Add rule 'magic_code'
kris.validator.rules.magic_code = function(value, param, el) {
    // value - current field value
    // param - parameter from HTML (e.g., "777")
    // el - DOM element (useful for attribute checks)
    
    return value === param;
};

// 2. Add error message (optional)
// If not added, default message is shown
window.KrisFormTranslateMessages.magic_code = "Wrong code. Expected: %s";

Function Signature

(value, param, element) => boolean
  • value: Field value (always string, unless checkbox/file).
  • param: Parameter string passed after colon (rule:param). null if no parameter.
  • element: Reference to DOM element input, select, or textarea.
  • return: Return true if validation passes. Return false for error.

JSON Dependencies

Object structure for form logic management.

Property Description
source name of the field to watch.
condition JS condition string. Example: value === 'admin'.
target Comma-separated list of target fields to affect.
action Action to perform if condition is True.
inverse_action Action to perform if condition is False.
message Message text (HTML allowed) next to the target.
confirm Text for the confirmation modal.

Action List

show / hide enable / disable required / optional check / uncheck clear set_value:X add_class:cls

Expression Syntax

Used in the condition field.

  • value — value of the source field.
  • fields.OtherName — value of another form field.
  • source.checked — (for checkboxes) true/false.
  • Operators: ===, !==, >, <, &&, ||.

1. Complex Logic

Example: Selecting "Company" shows Tax ID field and makes it required.

HTML Markup
<form id="form-complex">
  <select name="type" class="form-select">
    <option value="person">Individual</option>
    <option value="company">Company</option>
  </select>

  <!-- Tax Container (hidden by default) -->
  <div id="cont_tax" class="d-none mt-3">
    <label>Tax ID</label>
    <input name="tax" class="form-control" data-validator="numeric,len:10">
    <div class="invalid-feedback"></div>
    <!-- Place for dependency message -->
    <div class="field-status-message small text-primary"></div>
  </div>
</form>
JavaScript
const rules = [
  {
    "source": "type", "condition": "value === 'company'",
    "action": "show", "inverse_action": "hide", "target": "tax"
  },
  {
    "source": "type", "condition": "value === 'company'",
    "action": "required", "inverse_action": "optional", "target": "tax",
    "message": "Tax ID is required for legal entities"
  }
];
new KrisForm(document.getElementById('form-complex'), { dependencies: rules });

2. Modal Confirmations

The library supports action confirmation. You can use Bootstrap or your own markup (Vanilla).

Bootstrap 5
Vanilla JS
HTML (End of body)
<!-- For Bootstrap: Standard markup -->
<div class="modal fade" id="modal-bs" tabindex="-1">
  <div class="modal-dialog">
    <div class="modal-content">
      <div class="modal-body">
        <p class="confirm-message"></p>
      </div>
      <div class="modal-footer">
        <button class="btn btn-secondary btn-cancel">Cancel</button>
        <button class="btn btn-primary btn-confirm">OK</button>
      </div>
    </div>
  </div>
</div>

<!-- For Vanilla: Custom markup -->
<div id="modal-vanilla" style="display:none">
  <div class="my-custom-dialog">
    <p class="confirm-message"></p>
    <button class="btn-cancel">No</button>
    <button class="btn-confirm">Yes</button>
  </div>
</div>
JavaScript
// Config for Bootstrap form
new KrisForm(document.getElementById('form-bs'), {
  modalId: 'modal-bs', // Modal ID
  dependencies: [{
    source: "bs_sw", condition: "checked === true", 
    action: "enable", inverse_action: "disable", target: "bs_data",
    confirm: "Enable admin mode?" // Confirmation text
  }]
});

// Config for Vanilla form
new KrisForm(document.getElementById('form-vanilla'), {
  modalId: 'modal-vanilla',
  dependencies: [{
    source: "v_sw", condition: "checked === true", 
    action: "enable", inverse_action: "disable", target: "v_data",
    confirm: "Delete archive?"
  }]
});

3. Localization (Error Translation)

You can globally override error texts via the window.KrisFormTranslateMessages object. Click buttons below and try invalid inputs to see differences.

HTML
<form id="form-i18n">
  <label>Age</label>
  <input type="number" name="age" data-validator="required,min:18">
</form>
JavaScript
// Global Dictionary Setup
window.KrisFormTranslateMessages = {
  required: "This field is mandatory",
  min: "Minimum value: %s"
};

// Dynamic language switch (example)
function setLanguage(lang) {
  if (lang === 'en') {
    window.KrisFormTranslateMessages = { required: "Required", min: "Min: %s" };
  } else {
    window.KrisFormTranslateMessages = { required: "Обязательно", min: "Мин: %s" };
  }
}

new KrisForm(document.getElementById('form-i18n'));
